Data aggregations

Additional aggregations allow data to be filtered according to the customer's wishes.

  1. File Generation Language > PIC
  2. Available segments > PIC
  3. Available markets
  4. Currency as configured in ERP or NAV
  5. Aggregation by location
  6. File schedules - when files should be generated, e.g.:
    • Daily Schedule A
    • Once a week schedule B1 Monday B2 Tuesday …….
    • Once a month C+day of the month (e.g. 1 on every first day of the month)
    • please note that the ProductInformation and WholesalePricing files must have the same schedule set
    • in the Stock file you can set a schedule to refresh several times a day (every 2-3 hours)
  7. Visibility of the core index in the file
    • without core index visibility
    • with core index visibility
  8. Aggregation by leaves of the e-Cat catalog tree
    • i.e. if the customer wants only pads, tires, rims, etc.
  9. Aggregation by commodity status
  10. Aggregation after TecDoc
    • we set it to true/false depending on whether the customer wants only products with a TecDoc number or the entire offer
  11. Possibility to set the file to be compressed (*.zip) or not
  12. Possibility to choose the name of the generated file (a suffix with a date added every day or a fixed file name). Example: The default file name is created according to the following scheme: <File_Type_Year_Month_Day>.csv, e.g. Stock_2022-05-30.csv, after compression the file will be named Stock_2022-05-30.csv.zip
  13. Possibility to set file overwrite, i.e. the file will be in Stock.csv format and after compression Stock.csv.zip
